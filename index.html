The Concept

💡 An AI Second Brain is a personal knowledge system that sits between you and AI. Instead of starting every conversation from zero, the brain remembers your context — your clients, your style, your patterns, your tools — and compounds that knowledge with every interaction.

Think of it as three concentric rings:

🧬
Ring 1: Identity
Who you are. Your voice, vocabulary, communication style, domain expertise. The brain writes emails that sound like you, creates content in your tone, and applies your professional frameworks automatically.
📚
Ring 2: Knowledge
What you know. Client histories, meeting notes, research reports, contact details, market analysis. 520+ markdown files organized by domain — all searchable, all version-controlled.
⚡
Ring 3: Capability
What you can do. 56 slash commands, 6 specialized agents, 35 project skills, and 11 MCP integrations that connect to Outlook, WhatsApp, LinkedIn, Notion, Google Workspace, Todoist, and more.
The key insight: each task you complete makes the next one faster. The /learn command extracts patterns from completed work and stores them in memory. Over time, proposals get sharper, emails get more natural, and research gets deeper — because the brain has seen your work before.

How It Works

The Core Loop

/plan  →  /work  →  /review  →  /learn
📋 /plan
Break a complex task into 5 steps of ~10 minutes each. The brain considers your calendar, priorities, and past patterns.
⚙️ /work
Execute the plan step by step. Each step loads only the context it needs. The brain tracks progress and adapts if you get blocked.
🔍 /review
Quality check with multi-perspective validation. Catches gaps, inconsistencies, and AI-slop before you ship.
🧠 /learn
Extract patterns, update memory, log what worked. This is the compounding step — it makes future work faster.

Architecture

🏗️ The Second Brain is a markdown-first knowledge system orchestrated by Claude Code. Five layers, each serving a distinct purpose — from persistent storage to user-facing commands.

Layer 1: Persistent Storage

📁 Markdown Files

520+ files in memory/ organized by domain. Plain text, version-controlled in Git, human-readable. This is where all knowledge lives.

🗄️ Supabase

9 PostgreSQL tables (clients, contacts, meetings, kanban_cards, action_items, sessions, learning_events, research_reports, follow_ups). Dual-write: markdown first, Supabase second.

💾 Auto-Memory

MEMORY.md persists operational knowledge across sessions — API patterns, tool configs, contact lookup rules. 200-line limit, kept concise.

Layer 3: Intelligence

⚡ 35 Project Skills

Business skills that define how to do things: email triage, LinkedIn posting, meeting processing, market research, WhatsApp management, kanban board, training creation, PDF/PPTX handling. Each is a SKILL.md file in .claude/skills/.

🤖 6 Agents

Specialized personas in .claude/agents/: chief-of-staff, email-agent, content-creator, market-researcher, research-depth-analyzer, claude-code-specialist.

Layer 4: Automation (Headless Pipelines)

🔄 claude -p runs Claude non-interactively. Scripts scheduled via launchd execute on autopilot.

  • morning-briefing.sh — weekday 7am, generates daily briefing
  • inbox-zero.sh — automated email processing
  • Uses --allowedTools for least-privilege, --model sonnet for cost efficiency

Layer 5: User Interface

🎯 56 slash commands are the primary UI. You type /command and the brain executes. This GitHub Pages dashboard provides a visual overview. Supabase powers structured data storage and queries.

🔒 The Second Brain handles sensitive client data, email credentials, and API tokens. Security is layered across hooks, permissions, and operational practices.

Supply Chain Safety

  • All npx MCP packages version-pinned (e.g., @microsoft/workiq@0.2.8)
  • API tokens stored in macOS Keychain, never in config files
  • Supabase PAT accessed via Keychain wrapper script
  • skipDangerousModePermissionPrompt removed from global settings
  • Sandbox mode enabled with auto-allow for Bash
  • MCP wrapper stderr logs at /tmp/mcp-{name}.log for debugging
  • mcpsec baseline score: 77/100 (2026-02-27)
